Casey Reas created three unique structures—text descriptions outlining dynamic relations between elements—which were then implemented in association with Jared Tarbell, Robert Hodgin, and William Ngan. Each of them interpreted the textual structures to derive twenty-six software artworks. > "The catalyst for this project is the work of Sol LeWitt, specifically his wall drawings. I had a simple question: 'Is the history of conceptual art relevant to the idea of software as art?'" explains Reas. "I began to answer the question by implementing three of Lewitt's drawings in software and then making modifications. After working with the LeWitt plans, I created three structures unique to software. These software structures are text descriptions outlining dynamic relations between elements. They develop in the vague domain of image and then mature in the more defined structures of natural language before any thought is given to a specific machine implementation. Twenty-six pieces of software derived from these structures were written to isolate different components of software structures including interpretation, material, and process. For each, you may view the software, source code, and comments." {Software} Structures was restored in 2016 to work well with contemporary web browsers. The original software was created with Processing, which depends on Java applets that are no longer widely supported in web browsers. The updated version uses P5.js (a descendent of Processing using JavaScript) to be more accessible. - [http://whitney.org/Exhibitions/Artport/Commissions/SoftwareStructures](http://whitney.org/Exhibitions/Artport/Commissions/SoftwareStructures) ## Explorations of Sol LeWitt's wall drawings using P5.js ### P5 Editor The p5.js Web Editor is an online platform for learning to code in a visual way. Every piece of data is stored in it’s file by the key. It’s easy to find a file by it’s name or add/remove a file. - [https://javascript.info/object](https://javascript.info/object) ### Object literal syntax Object literals are defined using the following syntax rules: - [http://www.dyn-web.com/tutorials/object-literal/](http://www.dyn-web.com/tutorials/object-literal/) - A colon separates property name from value. - A comma separates each name-value pair from the next. - There should be no comma after the last name-value pair. ## P5.js example solutions - [Link](javascript.html)
